Warning!

Some of these levels could be exploited by a mailicious user to generate a hazardous response. None of the responses should include attachments or binary objects.

Use at your own risk!

The Goal

A great way to learn about computer security is to put on a white hat and learn what it takes to exploit a computer system. But, application owners usually frown upon people trying to break into their systems. As an aternative to breaking the law, this simple game provides a safe place for users to learn how to exploit a web based API and learn about the types of threats for which an unprotected API is vulnerable.

What You'll Need

All you need to play is an HTTP client that can support GET, PUT, POST and DELETE. For example, cURL, Postman, SOAP UI and RESTClient can all be used.

How to Play

  • Start at level 0 and exploit an API to get the location of the next level's API. Repeat until you reach the end of the game.
  • Each level presents a unique challenge and the exploits get more difficult as you progress.
  • Send an HTTP GET to /starthere and get started.

The Rules

  • Only HTTP requests are allowed
  • No shell, malware or man in the middle attacks are allowed
  • No denial of service attackas are allowed
  • Don't break the server just because you can - please play nice.
  • Try your best to use hints sparingly.
